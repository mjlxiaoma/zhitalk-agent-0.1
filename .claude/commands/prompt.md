# 提示词优化助手

优化 zhitalk-agent 项目的 AI 提示词。

## 输入参数
- $ARGUMENTS: 要优化的提示词位置或类型
  - `system` - 系统提示词 (`lib/ai/prompts.ts`)
  - `classify` - 分类提示词 (`lib/ai/agent/classify.ts`)
  - `agent:名称` - 特定代理的提示词

## 提示词位置

- **主提示词**: `lib/ai/prompts.ts`
  - `regularPrompt` - 通用系统提示词
  - `artifactsPrompt` - Artifact 使用指南

- **分类提示词**: `lib/ai/agent/classify.ts`
  - 用于意图分类的提示词

- **代理提示词**: `lib/ai/agent/*.ts`
  - 各个代理的专用提示词

## 优化原则

### 1. 清晰性
- 明确角色定位
- 清晰的任务描述
- 具体的输出要求

### 2. 结构化
- 使用 markdown 格式
- 分段组织内容
- 适当使用列表

### 3. 约束性
- 设定行为边界
- 防止角色越界
- 处理边缘情况

### 4. 效率
- 避免冗余信息
- 减少 token 消耗
- 保持核心指令

## 优化流程

1. 阅读当前提示词
2. 分析存在的问题
3. 参考最佳实践
4. 提出优化方案
5. 验证优化效果

## 提示词模板

### 系统提示词模板
```
你是 [角色名称]，[角色描述]。

## 核心职责
- 职责 1
- 职责 2

## 工作流程
1. 步骤 1
2. 步骤 2

## 输出要求
[格式和内容要求]

## 注意事项
[边界和限制]
```

### 分类提示词模板
```
分析用户消息，判断其意图类型。

## 意图类型
- type_a: 描述
- type_b: 描述

## 判断规则
[规则说明]

## 输出格式
仅输出意图类型，不要解释。
```

## 评估指标

优化后检查：
- [ ] 任务完成率是否提升
- [ ] 响应质量是否改善
- [ ] token 使用是否减少
- [ ] 边缘情况处理是否更好
